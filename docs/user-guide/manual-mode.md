# Manual 模式（半自动）

> **每步审批、精细控制** —— 适合需要逐环节调整的创作者。

---

## 一、与 Autonomous 模式的区别

| 维度                 | Autonomous        | Manual               |
| -------------------- | ----------------- | -------------------- |
| **用户参与度**       | 零                | **高**（每步）       |
| **Quality Gate**     | 自动              | ⚠️ 仅提示            |
| **Self-Review Loop** | 启用              | ❌ 关闭              |
| **断点续传**         | 启用              | ❌ 不支持            |
| **适用场景**         | 批量、快速        | 精细、定制           |
| **预估时间**         | 15-30 min（短篇） | 数小时（取决于细调） |

---

## 二、使用流程

### 步骤 1：新建项目（Manual 模式）

打开 Story Weaver → 点击「**新建项目**」→ 选择「**Manual 模式**」

### 步骤 2：选择输入文件

支持 `.txt` `.docx` `.md` `.pdf`，或直接粘贴文本。

### 步骤 3：逐步执行

你会看到 10 个步骤**依次排列**，每步可独立操作：

```
┌──────────────────────────────────────────────────┐
│ 步骤 1/11: 导入                                  │
│                                                  │
│  状态：✅ 已完成                                  │
│  产物：12 个章节、3.4 万字                       │
│  [重新执行] [查看详情]                            │
├──────────────────────────────────────────────────┤
│ 步骤 2/11: 分析                                  │
│                                                  │
│  状态：🔄 进行中  65%                             │
│  [取消] [查看实时日志]                            │
├──────────────────────────────────────────────────┤
│ 步骤 3/11: 脚本                                  │
│                                                  │
│  状态：⏸️ 等待执行                                │
│  [执行此步] [跳过]                                │
└──────────────────────────────────────────────────┘
```

### 步骤 4：审批 / 编辑

每步完成后，你可以：

| 操作            | 说明                         |
| --------------- | ---------------------------- |
| **✅ 通过**     | 进入下一步                   |
| **✏️ 编辑**     | 修改产物后再进入下一步       |
| **🔄 重新执行** | 用不同参数重跑此步           |
| **⏭️ 跳过**     | 直接进入下一步（用默认结果） |

---

## 三、可编辑的产物

不同步骤产物的**可编辑程度**不同：

| 步骤    | 产物          | 可编辑                |
| ------- | ------------- | --------------------- |
| 1 导入  | 章节切分      | ✅ 增删章节、修改标题 |
| 2 分析  | 人物/场景清单 | ✅ 添加/删除人物      |
| 3 脚本  | JSON 剧本     | ✅ 完全可编辑         |
| 4 角色  | 角色设定卡    | ✅ 修改外观/服装      |
| 5 场景  | 场景规划      | ✅ 修改镜头           |
| 6 分镜  | 分镜脚本      | ✅ 修改画面描述       |
| 7 渲染  | 关键帧        | ⚠️ 仅"重新渲染"       |
| 8 剪辑  | 视频片段      | ⚠️ 仅"重新剪辑"       |
| 9 配音  | 音频          | ⚠️ 切换 voice 重生成  |
| 10 字幕 | SRT/VTT       | ✅ 完全可编辑         |
| 11 导出 | MP4           | ⚠️ 仅"重新导出"       |

## 四、典型工作流

### 4.1 修改角色外观

```
步骤 4 完成 → 角色: 李明
   │
   ▼
点击"编辑"
   │
   ▼
修改：李明 短黑发 → 棕色短发
   │
   ▼
保存
   │
   ▼
系统提示："修改将影响分镜 1.2 / 3.5 / 7.1 是否重新渲染？"
   │
   ├─ [是] → 自动重新渲染相关分镜
   └─ [否] → 保留现有分镜
```

### 4.2 调整分镜

```
步骤 6 完成 → 分镜 5.2 画面
   │
   ▼
发现运镜不对（应该是"推"而不是"摇"）
   │
   ▼
点击"编辑" → 修改 camera.movement = 'push'
   │
   ▼
点击"重新生成参考图" → 仅此分镜重渲染
```

### 4.3 跳过自审循环

Manual 模式默认**关闭 Self-Review Loop**，因此：

- 不会自动重试
- 不会优化 Prompt
- 评分仅供参考，**最终决定权在你**

如需在 Manual 模式启用自审：

```
设置 → 高级 → Manual 模式启用 Self-Review → [开关]
```

## 五、适用场景

### ✅ 适合 Manual 模式

- **商业项目** — 客户要求每一帧都精确
- **风格统一** — 整部漫剧统一美术风格
- **角色定制** — 主角在每集中的细节调整
- **实验性创作** — 探索不同 prompt 效果
- **预算紧张** — 避免自动重试产生额外 API 费用

### ❌ 不适合 Manual 模式

- 大量短剧批量生产 → 改用 Autonomous
- 完全信任 AI → 改用 Autonomous
- 无时间逐个调整 → 改用 Autonomous
